# ReadQuest 미리 생성 콘텐츠 · 문제/답 검증 요약

## 검증 기준
- **O/X 퀴즈**: `correctAnswer`가 'O' 또는 'X', `question`·`explanation` 존재
- **객관식/빈칸채우기/같은 상황 다른 말**: `correctIndex`가 `options` 인덱스 범위(0 ~ options.length-1) 내
- **순서맞추기**: `items` 배열 순서가 정답 순서
- **감정 흐름 계단**: `correctPlacement` 길이 = `steps` 길이, 각 값이 `items` 인덱스
- **따뜻해진 이유 찾기**: `realReasonIndices`가 `reasons` 인덱스
- **말-결과 연결**: `correctPairs`가 `leftItems`·`rightItems` 인덱스 쌍

### 1.5 감정 흐름 계단 (10문항)
- 각 문항: 5단계(steps), 5개 사건 카드(items), `correctPlacement` 배열로 단계별 정답 카드 인덱스 지정.  
- 1~2번: [0,1,2,3,4]. 3번: [0,1,1,2,2]. 4~10번: 일부 단계에만 정답 배치 또는 [1,1,2,3,4] 등.  
- **참고**: 3번 등 `correctPlacement`에 중복 인덱스가 있으면 “한 카드는 한 단계에만” 배치하는 UI에서는 일부 카드가 정답으로 쓰이지 않음(오답용 카드로 동작).

### 1.6 따뜻해진 이유 찾기 (10문항)
- 각 문항: `reasons` 5개, `realReasonIndices`로 진짜 이유 인덱스만 지정(1~3개).  
- 인덱스는 모두 0~4 범위 내.

## 2. 몽몽 숲의 박쥐 두마리 (book-mongmong)

- `correctAnswer`는 모두 'O' 또는 'X'.

### 2.2 객관식 (17문항)
- 정답 인덱스: 2,0,0,1,0,1,1,0,1,1,1,1,1,1,0,1,0.  
- 옵션 2~4개, `correctIndex` 모두 유효.

### 2.3 순서맞추기 (10문항)
- 정답 순서는 각 문항의 `items` 배열 순서.

### 2.5 말-결과 연결 (10문항)
- 각 문항: `leftItems` 4개, `rightItems` 4개, `correctPairs` [[0,0],[1,1],[2,2],[3,3]].  
- 왼쪽 i번 ↔ 오른쪽 i번이 정답 쌍.

### 2.6 같은 상황 다른 말 (10문항)
- 각 문항: `situation`, `options` 2개, `resultTexts` 2개, `correctIndex` 0(첫 번째 선택이 정답).  
- 정답은 모두 “다정한/긍정적 말” 선택지.
